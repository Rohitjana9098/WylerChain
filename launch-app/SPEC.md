# Project Specification: WylerChain Premium Launch App

## Overview
Recreate the WylerChain reference website as a feature-parity single-page Web3 product app. The app must have the same end-to-end functional structure but with original branding, copy, and assets, using the "WylerChain Premium Launch UI" design system.

## Core Objectives
- Feature-parity with the reference Web3 product app.
- Single-page application (SPA) behavior with internal screen switching.
- Reusable component architecture.
- Modular product layers (8 modules).

## Product Layers
1. **Public Landing Screen**: Hero, Intro, Feature Grid, CTAs (Launch/Demo), Product messaging.
2. **Login Screen**: Social login methods, automatic wallet creation, back action.
3. **Authenticated Dashboard**: Top nav (Wallet, Staking, Creators, NFTs, Explore), Connected state.
4. **Wallet Module**: Total balance, Asset breakdown, Network display, Send/Receive/Bridge/Stake actions, KPI cards, Transactions list.
5. **Staking Module**: Staked balance, APY, Status, Reward summary, Lock duration, Projected rewards, Actions (Stake/Claim/Unstake), Tiers (Bronze to Platinum), Validator list.
6. **Creator Module**: Creator grid/feed, Profile cards, Tip flow, Monetization positioning.
7. **NFT Module**: NFT grid, Card with details, Mint NFT action, Creator-linked economy.
8. **Explore Module**: Discovery section for creators, NFTs, and ecosystem activity.

## General Requirements
- **Design System**: Use "WylerChain Premium Launch UI" (Obsidian Void aesthetic).
- **Interactions**: Tab switching, modals/side-panels for actions, responsive layouts.
- **Data**: Mock data for creators, NFTs, validators, balances, and transactions.
- **State**: Centralized app state for login, navigation, and user data.
